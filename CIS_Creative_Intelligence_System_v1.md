# **CIS — Creative Intelligence System (v1)**

**A Feedback-Driven Knowledge & Production System**

---

# **0\. 🧭 WHAT THIS SYSTEM IS (Plain Explanation)**

CIS is a system that turns **raw creative material** (files, videos, ideas) into **structured, reusable knowledge**, and then uses that knowledge to **produce real outputs** (design, animation, sound, etc.).

It works by combining:

* machine interpretation (AI)  
* human correction (you)  
* structured storage (knowledge records)  
* repeatable execution (commands)

---

## **🔁 The Core Idea**

Instead of defining everything upfront, CIS:

👉 **discovers structure from your real material**  
 👉 **improves through correction and reuse**  
 👉 **becomes more accurate over time**

---

## **🔄 The Core Loop**

Source Material   
→ AI Interpretation   
→ Structured Knowledge   
→ Human Review   
→ Storage   
→ Reuse in Projects   
→ Output Creation   
→ Feedback → Improved System

# **2\. 🧩 CORE COMPONENTS (What the system is made of)**

---

## **2.1 🗂 Project System (Center of Everything)**

A **Project** is the main container.

It holds:

* sources (files)  
* knowledge records  
* workflow state  
* outputs

👉 All work happens inside a project.

---

## **2.2 📥 Source System (Input Layer)**

Sources \= raw material:

* PDFs  
* images  
* videos  
* folders

Each source gets a:

👉 **Source Manifest**

* tracks identity  
* tracks processing state  
* controls execution

---

## **2.3 ⚙️ Execution System (The Engine)**

Execution \= how work is actually done.

Every step must define:

* input  
* process  
* output  
* validation  
* failure rules  
* state change

---

### **Example (real)**

cis extract \<source\_id\>

Must produce:

* structured output (not text)  
* required fields  
* valid tags  
* uncertainty markers

If not → FAIL

---

## **2.4 🧠 Intelligence System (AI Layer)**

AI is used to:

* extract meaning from material  
* propose structure  
* generate knowledge records

BUT:

👉 AI does NOT decide truth  
 👉 AI proposes — human confirms

---

### **Multi-pass processing:**

1. OCR (text)  
2. Visual interpretation  
3. Merge (REQUIRED)  
4. Structure output

---

## **2.5 📚 Knowledge System (Memory Layer)**

The only valid knowledge unit:

👉 **Knowledge Record**

It must include:

* source  
* type  
* content  
* tags  
* uncertainty  
* status

---

### **Important Rule**

❌ Raw files are NOT knowledge  
 ✅ Only validated records are knowledge

---

## **2.6 🔄 Workflow System (Movement of Work)**

Workflow defines the path:

Idea → Source → Processing → Knowledge → Output → Reuse

It is:

* flexible  
* non-linear  
* project-based

---

### **Panels:**

* Review Panel → human correction  
* Output Panel → final knowledge
